# SECTION 1 ‚Äî INTRODUCTION

## What is This Software?

This is a **Distribution Management System (DMS)** designed specifically for FMCG (Fast Moving Consumer Goods) distribution businesses. It is a complete business management tool that helps you:

- Manage your products and inventory
- Track all your customers and vendors
- Create and manage sales orders
- Record purchases from suppliers
- Handle all payments (money received and money paid)
- Track expenses
- Generate business reports
- Keep accurate accounting records

## Who Should Use This Software?

This software is designed for:

| User Type | What They Do |
|-----------|--------------|
| **Distributor (Owner/Admin)** | Business owner who oversees everything |
| **Computer Operator** | Office staff who enters data, manages inventory, records payments |
| **Order Booker** | Field salesmen who visit customers, take orders, collect cash |
| **Customer** | Shop owners who can view their orders and account balance |

## What Problems Does This Software Solve?

| Problem | How This Software Helps |
|---------|------------------------|
| Paper records getting lost | All data is stored safely in the computer |
| Manual calculation errors | System calculates totals automatically |
| Difficulty tracking payments | Complete payment history for every customer and vendor |
| Not knowing current stock | Real-time stock levels always available |
| Messy accounts | Automatic double-entry bookkeeping |
| No visibility of profits | Instant profit and loss reports |
| Order bookers working without tracking | Every order is recorded with the order booker's name |
| Customer balance disputes | Complete ledger available for any customer |

## Benefits of Using This Software

1. **Save Time** - No more manual register writing
2. **Reduce Errors** - System does all calculations
3. **Better Control** - Track every transaction with user name and time
4. **Real-time Information** - Know your stock, sales, and payments instantly
5. **Professional Invoices** - Generate printed invoices and receipts
6. **Financial Reports** - Get Trial Balance and Profit & Loss anytime
7. **Mobile Access** - Works on phones, tablets, and computers

### How Login Works

1. Every user has a **unique username** and **password**
2. When you login, the system checks your username and password
3. If correct, you are taken to your **Dashboard**
4. If wrong, you see an error message
5. The system remembers you for 7 days (you don't need to login again each day)

### Password Rules

Your password must:
- Be **at least 6 characters** long
- Be kept **private** - never share with anyone
- Be **changed** if you suspect someone knows it

### Security Basics

| DO | DON'T |
|----|-------|
| ‚úÖ Keep your password secret | ‚ùå Share login details with others |
| ‚úÖ Logout when leaving the computer | ‚ùå Use easy passwords like "123456" |

# SECTION 5 ‚Äî ACCOUNTING BASICS (SIMPLE LANGUAGE)

## What is Debit and Credit?

Think of your business like a story with two sides:

**DEBIT (Left Side):**
- Money coming IN to a cash/bank account
- Goods you buy (assets increasing)
- Expenses you pay

**CREDIT (Right Side):**
- Money going OUT of a cash/bank account
- Sales you make (income)
- Money you owe to others (liability)

### Simple Example:

| Transaction | Debit (Left) | Credit (Right) |
|------------|--------------|----------------|
| You sell goods worth Rs.1000 | Cash Rs.1000 | Sales Rs.1000 |
| You buy goods for Rs.500 | Purchases Rs.500 | Cash Rs.500 |
| Customer owes you Rs.1000 | Customer Account Rs.1000 | Sales Rs.1000 |

**Remember:** Every transaction has BOTH a debit and a credit. They must be equal!

## What is Ledger?

A **Ledger** is like a diary for each person or account showing all transactions.

**Example: Customer "ABC Store" Ledger**

| Date | Description | Debit | Credit | Balance |
|------|-------------|-------|--------|---------|
| 1st Jan | Opening Balance | - | - | Rs. 0 |
| 5th Jan | Invoice #101 | Rs. 5,000 | - | Rs. 5,000 |
| 10th Jan | Payment Received | - | Rs. 3,000 | Rs. 2,000 |
| 15th Jan | Invoice #102 | Rs. 3,000 | - | Rs. 5,000 |

This tells you that ABC Store currently owes you **Rs. 5,000**.

## What is Trial Balance?

A **Trial Balance** is a summary of ALL accounts on a specific date.

Think of it like a scale:
- **Left side (Debits)** must equal **Right side (Credits)**
- If they don't match, something is wrong

**Example Trial Balance:**

| Account | Debit | Credit |
|---------|-------|--------|
| Cash | Rs. 50,000 | - |
| Bank | Rs. 200,000 | - |
| Inventory | Rs. 150,000 | - |
| Customers (Receivable) | Rs. 100,000 | - |
| Vendors (Payable) | - | Rs. 75,000 |
| Owner's Capital | - | Rs. 300,000 |
| Sales | - | Rs. 200,000 |
| Purchases | Rs. 60,000 | - |
| Expenses | Rs. 15,000 | - |
| **TOTAL** | **Rs. 575,000** | **Rs. 575,000** |

‚úÖ Both sides equal = Books are balanced!

## What is Profit & Loss?

**Profit & Loss** statement shows if your business made money or lost money.

**Simple Calculation:**
```
Sales (Income)
MINUS
Cost of Goods Sold
MINUS
Expenses
= Profit (or Loss)
```

**Example:**

| Description | Amount |
|-------------|--------|
| Total Sales | Rs. 500,000 |
| Cost of Goods Sold | (Rs. 350,000) |
| **Gross Profit** | Rs. 150,000 |
| Rent Expense | (Rs. 20,000) |
| Salary Expense | (Rs. 30,000) |
| Electricity | (Rs. 5,000) |
| Transport | (Rs. 10,000) |
| **Net Profit** | **Rs. 85,000** |

You made **Rs. 85,000 profit** this period!

## Why Balancing is Important

1. **Find Errors:** If books don't balance, there's a mistake somewhere
2. **Trust:** Balanced books mean your records are likely correct
3. **Legal:** Tax authorities expect balanced records
4. **Decisions:** You can't make good decisions with wrong numbers
5. **Fraud Detection:** Imbalanced books may indicate fraud

## Common Accounting Mistakes to Avoid

| Mistake | How to Avoid |
|---------|-------------|
| Entering wrong amount | Always double-check numbers before saving |
| Selecting wrong customer | Read the customer name carefully |
| Forgetting to record payment | Enter receipts immediately when received |
| Entering date wrong | Check date, especially at month start/end |
| Double entry | Check if similar entry exists before adding |
| Wrong payment method | Verify if it's cash, cheque, or bank transfer |
| Missing vendor invoice number | Always get and enter the vendor's bill number |
| Not recording expenses | Enter every expense, even small ones |

## Trial Balance

### What It Shows:
- Summary of ALL accounts on a specific date
- Total debits and total credits (must be equal!)
- Grouped by account type

### When to Use:
- End of month/year closing
- Before preparing Profit & Loss
- To check if books are balanced
- Finding errors in accounts

### How to Read It:

| Account Type | Example Accounts |
|-------------|------------------|
| **Assets** | Cash, Bank, Inventory, Receivables |
| **Liabilities** | Payables, Loans |
| **Equity** | Owner's Capital |
| **Income** | Sales |
| **Expenses** | Rent, Salaries, Transport |

**Check:** Total Debits = Total Credits (if not, there's an error!)

---

## Profit & Loss

### What It Shows:
- Total income (sales)
- Cost of goods sold
- All expenses
- Net profit or loss

### When to Use:
- Monthly performance review
- Quarterly business analysis
- Year-end tax calculation
- Business planning

### How to Read It:

**Section 1: Income**
- Shows total sales and other income
- This is money you earned

**Section 2: Cost of Goods Sold**
- Shows the cost of products you sold
- This reduces your gross profit

**Section 3: Gross Profit**
- Sales MINUS Cost of Goods = Gross Profit
- This is your profit before expenses

**Section 4: Operating Expenses**
- Rent, Salaries, Utilities, Transport, etc.
- These reduce your net profit

**Section 5: Net Profit/Loss**
- Gross Profit MINUS Expenses = Net Profit
- If positive: You made money! üéâ
- If negative: You lost money üòü

# SECTION 8 ‚Äî COMMON MISTAKES & TROUBLESHOOTING

## What to Do If Wrong Entry Added

### Wrong Order Entry:

**If order is still "Pending":**
1. Go to Orders
2. Find the order
3. Click on it
4. Click "Edit"
5. Make corrections
6. Save

**If order is already "Approved" or "Delivered":**
- Contact your Admin/Distributor
- They may need to create a credit note
- Or make adjusting entry
- Document the error in remarks

### Wrong Receipt Entry:

**If just entered (same day):**
1. Contact Admin immediately
2. Admin can delete/void the entry
3. Re-enter correctly

**If from previous days:**
- Do NOT delete (this affects accounting)
- Create an adjusting entry
- Or contact Admin for guidance

---

## What to Do If Stock Mismatch

### Steps to Resolve:

1. **Verify Physical Count:**
   - Count the product again
   - Have another person verify
   - Check all storage locations

2. **Check Recent Transactions:**
   - Were any orders delivered but not recorded?
   - Were any returns not entered?
   - Was there recent purchase not entered?

3. **Check for Damage/Loss:**
   - Look for damaged goods
   - Check for expired products
   - Investigate possible theft

4. **Make Adjustment:**
   - Go to Inventory ‚Üí Adjustment
   - Enter actual count
   - Select appropriate reason
   - Add detailed notes

5. **Report to Admin:**
   - If difference is large, inform management
   - Keep record of frequent mismatches

---

## What to Do If Payment Entered Wrongly

| Wrong Element | Solution |
|--------------|----------|
| Wrong amount | Create reverse entry + correct entry |
| Wrong customer | Contact Admin to void and re-enter |
| Wrong date | If minor, often acceptable; if major, adjust |
| Wrong payment method | Usually requires Admin correction |

**Steps:**
1. Note down what was wrong and what it should be
2. Contact Admin/Supervisor immediately
3. Don't make multiple entries trying to fix it yourself
4. Wait for guidance on how to correct

## Quick Formulas

**Customer Balance:**
```
Previous Balance + Sales - Receipts = Current Balance
```

**Vendor Balance:**
```
Previous Balance + Purchases - Payments = Current Balance
```

**Gross Profit:**
```
Sales - Cost of Goods Sold = Gross Profit
```

**Net Profit:**
```
Gross Profit - Expenses = Net Profit
```

**Stock After Transaction:**
```
Previous Stock + Purchases - Sales ¬± Adjustments = Current Stock
```
